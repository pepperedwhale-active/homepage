Converting LaTeX documents to HTML files using LaTeXML

We will discuss how to convert LaTeX files to HTML files using LaTeXML

Recently, arXiv introduced a new feature to view pdfs in html format . I started wondering how they implemented this, and came across the arXiv-vanity project, which is where this technology took birth. The creators explained that they used LaTeXML , a LaTeX to XML converter. I decided to use this library for my own home page, and you can see the pdf version alongside the HTML version of my notes in Notes/Posts. In this post, I will show how you can use this tool to convert your own PDFs to HTML as well using LaTeXML.

First, install LaTeXML. I am using debian, so i'd write:
sudo apt install latexml.
As of writing this, LaTeXML is still a very new tool, and as such, it does not mingle well with expl3. So if you have any packages that use expl3, i'd recommend you remove them. In my case, I was using newpx and newtx for font styles.
Now, if the .tex file you want to convert is called input.tex, type
latexml --destination=output.xml input
This will produce an XML file named output.xml.
Now, type:
latexmlpost --destination=output.html output.xml
This will produce the final .html output, converting your .tex file to .html output.

This was a very simple conversion without many customisations to get you started. In my case, I also used the --splitat=chapter option when calling latexmlpost to split my pdf into multiple chapters, as you can see in Module and Galois Theory. I also added some custom css styles, so that the output matches the color scheme of the rest of the site.
