Converting LaTeX documents to HTML files using LaTeXML (and why you should try it)

Recently, arXiv introduced a new feature to view pdfs in html format . I started wondering how they implemented this, and came across the arXiv-vanity project, which is where this technology took birth. The creators explained that they used LaTeXML , a LaTeX to XML converter. I decided to use this package for my own home page, and you can see the pdf version alongside the HTML version of my notes in Notes/Posts. In this post, I will show how you can use this tool to convert your own PDFs to HTML as well using LaTeXML.

Note: If you dont want to download this package, but want to try it out, go to https://latexml.mathweb.org/editor.

Coming back to the set up, first install LaTeXML. I am using debian, so i'd write:

sudo apt install latexml.

As of writing this, LaTeXML is still a very new tool, and as such, it does not implement some latex packages yet. So if you have any packages like that, i'd recommend you remove them. In my case, I was using newpx and newtx for font styles.
Now, if the .tex file you want to convert is called input.tex, type

latexml --destination=output.xml input

This will produce an XML file named output.xml.
Now, type:

latexmlpost --destination=output.html output.xml

This will produce the final .html output, converting your .tex file to .html output.

This was a very simple conversion without many customisations to get you started. In my case, I also used the --splitat=section option when calling latexmlpost to split my pdf at the \section level, as you can see in Algebraic Topology . I also added some custom css styles by using the --css=filename.css option, so that the output matches the color scheme of the rest of the site. Here is the documentation, for more details.

Why do this?

One answer could be that PDFs are just BAD. They are practically unusable on mobile devices(compared to web pages atleast). I say this from personal experience. One keeps wasting time moving around the document, squinting at it, etc. Although there are great PDF viewers for mobile, I still consider Web pages superior.

Another answer could be that this is exactly what the World Wide Web was designed for. That is, the WWW was initially created by academics to share their work. Web pages for general public use, as we know them today, began only around 1993, about 5 years after the creation of the WWW.

There are also tools like Obsidian , but I find them eating up a lot of ram, and I dont like the fact that files are stored as markdown files causing almost no room to apply customisations. Moreover, pages load much faster than if you used something like Obsidian.
