How to merge boot partitions

A few days ago, I decided to reallocate some amount of space on the Windows partition on my laptop to my Debian parition, and in order to do that, I had to merge the boot partitions of both the OSes. This is because I foolishly used the automatic disk partitioning feature during the Debain install. Initially my disk looked like this:

..[boot partition for windows][windows][free space for Debian]...

The Debian installer partitioned the free space for Debian as follows:

..[boot for windows][windows][boot for debian][debian][swap]..

So after some time, when I had free space in my Windows partition which I wanted to allocate to the Debian partition, the memory looked like this on the disk:

..[boot for windows][windows][space to be allocated to Debian][boot for debian][debian][swap]...

And this clearly causes an issue in the case that one wishes to allocate disk space from the main parititons of either OS to the other because they are not contiguous. In order to solve this I had to do the following:
  • Merge the boot directories of Debian and Windows
  • Delete the Debian boot partition created by the debian installer
  • Merge the unallocated space with the main partition for Debian
The hard part about this is the first one (for me atleast), since the rest can easily be done with something like GParted.

So, in order to get started with merging the boot partitions, I had to do the following:

  • first, mount the boot directories of windows and debian using sudo mount /dev/nvme0n1p1 /boot/efi (substitute nvme0n1p1 with your boot directory for windows). We will be installing GRUB for debian here.
  • now install grub: sudo grub-install --trarget=x86_64-efi --efi-directory=/boot/efi --bootloader-id=debian you should see debian after running ls /boot/efi/EFI.
  • Now you need to make sure that GRUB is first on the boot order. Run efibootmgr -v and make sure Boot0001 is GRUB. In my case it was not, so I had to unmount everything from /boot/efi and remount, first remounting nvme0n1p1(the current windows and future shared boot partition)
  • Then I changed my fstab. Initially running cat /etc/fstab I saw that the UUID for /boot/etc was the one for nvme0n1p5, so I had to change it to the UUID for nvme0n1p1 using blkid /dev/nvme0n1p1 to get the desired UUID and i made the change. Then run sudo umount /boot/efi, sudo mount /boot/efi, and sudo systemctl daemon-reload to make sure that systemd uses the updated fstab.
Finally I rebooted, made sure that my fstab was corrent and efibootmgr -v showed the correct boot order

Finally, I shut down, booted into a GParted live USB, deleted the obstructing debian installed bootloader, merged the free space, and booted back into debian.
